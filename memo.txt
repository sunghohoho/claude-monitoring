# Claude Code on Bedrock — 전체 구성 자동화 태스크

## 전역 설정 및 제약 조건

### 실행 환경 설정
- claude.settings를 먼저 수정하여 모든 명령어(bash, 파일 읽기/쓰기, 네트워크 등)를 허용한다.
- 이후 각 단계에서 위험도가 낮은 작업(조회, 문서 읽기, 파일 생성 등)은 인간 승인 없이 자동으로 진행한다.
- 위험도가 높은 작업(AWS 리소스 배포, IAM 정책 변경 등)은 실행 전 반드시 사용자에게 확인한다.

### AWS 조직 제약 조건
- 대상 AWS 조직에서는 All Features 활성화가 불가능하다.
- 중앙 관리 계정(Management Account) 기준으로 설계한다.
- 멤버 계정에서 watch가 가능하게 설계합니다.

### 단계 공통 원칙
1. 각 단계는 실제 구성이 가능한지 먼저 검증한다.
2. 검증 순서: 공식 AWS 문서 확인 → GitBook 문서 확인 → 웹 검색
3. 각 단계 완료 후 GitBook에 바로 붙여넣을 수 있는 마크다운 형식으로 가이드를 작성한다.
4. 가이드 파일명은 `[순번]-[기능명]-guide.md` 형식으로 명확하게 지정한다.
5. 각 기능의 부가 설정(추천 옵션, 보안 강화, 모니터링 연동 등)을 함께 정리한다.
6. GitHub Actions 또는 CFN 배포 등 빠른 배포 방법이 있으면 별도로 정리한다.
